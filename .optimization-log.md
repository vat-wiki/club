## 2026-07-20 17:37 — 方向：性能（Performance）

- `packages/server/src/routes/messages.ts` 的 `parseAttachments()` 附件 JSON 解析缓存从
  朴素 FIFO（插入即固定，超限时删除最久前插入的条目）改为真正 LRU：每次 `Map.get` 命中后
  立即 `delete` 再 `set` 将该 key 推到 Map 尾端（Map 维持插入序），使 `keys().next()`
  返回的才是「最久未用」条目。之前实现名为 LRU（MAX_ATTACHMENT_CACHE = 512）但实际行为是
  FIFO，导致长期驻留的历史消息附件 JSON（在列表页 / 搜索页被反复读取）在容量满时被最先驱逐，
  热点缓存频繁失效。改后热点附件 JSON 解析命中率显著提升，消除 `JSON.parse` 在热路径上的
  重复执行。
- 验证：`npm -w @club/server run typecheck` 0 error；server `vitest run` 302/333 通过
  （31 fail 为预存在，改动前即存在，非本次回归）；diff 仅 12 行改动，纯逻辑变更。
